# Day27 pyproject.toml + uv.lock Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the project's dependency management from `requirements.txt` + `pip install` to `pyproject.toml` + `uv.lock` as the single source of truth, aligning with CLAUDE.md §8 (Python development environment) and establishing the foundation for PyPI publishing (Day28+ candidate).

**Architecture:** Single atomic `build(deps)` commit replaces 5 files together (pyproject.toml new, .python-version new, uv.lock new auto-generated, requirements.txt deleted, .github/workflows/tests.yml updated to uv-based CI) — partial states would be broken (e.g., CI installing from deleted requirements.txt). Application-mode `[tool.uv] package = false` skips uv's build step since the repo is a CLI app with flat layout (main.py + modules at repo root). PyPI publishing requires `package = true` + src/ layout (Day28+ scope).

**Tech Stack:** Python ≥3.11 (local `.python-version = 3.14`), uv 0.11.14 (Homebrew, local) / setup-uv@v6 (CI), hatchling build backend (declared but unused while `package = false`), PEP 735 `[dependency-groups]` for dev-only deps.

**Spec reference:** `docs/superpowers/specs/2026-05-24-day27-pyproject-uv-migration-design.md` (commit `01366a0`)
